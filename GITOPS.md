# OpenShell GitOps Deployment

Full GitOps deployment for OpenShell on OpenShift. Everything is
declared in Git and reconciled by ArgoCD — including Vault itself.

## Architecture

```
deploy/gitops/
├── app-of-apps.yaml                # Single entry point — deploys everything
├── argocd-app.yaml                 # OpenShell Application (Helm + kustomize)
├── infra/
│   ├── vault/
│   │   └── argocd-app.yaml         # Vault server (Helm chart via ArgoCD)
│   └── vault-config/
│       ├── argocd-app.yaml         # Vault config Application
│       └── manifests/
│           └── vault-config-job.yaml  # K8s auth, policies, JWT keygen, vault-token
├── base/
│   ├── kustomization.yaml          # Base kustomize resources
│   ├── namespace.yaml              # openshell namespace
│   ├── scc-binding.yaml            # OpenShift SCC for sandbox pods
│   └── values-base.yaml            # Shared Helm values (no secrets)
└── overlays/
    └── openshift-dev/              # Dev overlay (no TLS, ESO for JWT)
        ├── kustomization.yaml
        ├── values-patch.yaml
        └── external-secrets.yaml   # SecretStore + ExternalSecret
```

## Sync Wave Ordering

ArgoCD deploys in this order via sync waves:

1. **Wave 0**: Vault server (Helm chart)
2. **Wave 1**: Vault configuration (K8s auth, policies, JWT key generation, vault-token Secret)
3. **Wave 2**: OpenShell (namespace, SCC, SecretStore, ExternalSecret, Helm chart)

## Secret Management

**No secrets are stored in Git.** All sensitive values are generated by the
Vault configuration Job and served via External Secrets Operator:

- JWT signing keys: generated by vault-config Job, stored in Vault at `openshell/jwt-keys`
- Vault access token: created by vault-config Job as K8s Secret `vault-token` in `openshell` ns
- ESO syncs JWT keys from Vault into K8s Secret `openshell-jwt-keys`

## Prerequisites

1. OpenShift 4.14+ with OpenShift GitOps (ArgoCD) operator installed
2. External Secrets Operator installed
3. Agent Sandbox CRDs and controller (one-time cluster bootstrap)

## Deployment (Full GitOps)

One command deploys the entire stack — Vault, configuration, and OpenShell:

```shell
# 1. One-time: Install Agent Sandbox CRDs (cluster-wide prerequisite)
oc apply -f https://github.com/kubernetes-sigs/agent-sandbox/releases/latest/download/manifest.yaml

# 2. Deploy the App-of-Apps (ArgoCD reconciles everything else)
oc apply -f deploy/gitops/app-of-apps.yaml
```

ArgoCD then automatically:
- Deploys Vault (wave 0)
- Configures Vault auth + policies + generates JWT keys (wave 1)
- Deploys OpenShell with ESO-managed secrets (wave 2)

No `helm install`, no `oc exec`, no imperative commands after step 2.
