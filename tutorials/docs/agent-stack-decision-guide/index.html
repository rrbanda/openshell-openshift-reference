Choose the right agent stack

OpenClaw, Hermes, Google ADK, LangGraph, and “autonomous agents” are not rivals on one axis. Decide the control pattern, then whether you adopt a product runtime or build with a framework, then isolation. Wrong layer comparisons produce wrong picks.

01 · Mental model

Four layers, four decisions

Treat each option as belonging to a layer. A complete system often combines one choice from several layers. Comparing LangGraph to OpenClaw without saying which layer you are deciding is the most common failure mode.

Layer 0
Pick one pattern Workflow Hybrid Autonomous loop Always-on presence
THEN ↓ pick runtime or framework (usually not both as the product)
Layer 1 or 2
One primary path OpenClaw / Hermes / Deep Agents or LangGraph / ADK / BYOA
THEN ↓ choose isolation / packaging
Layer 3
Pick isolation NemoClaw OpenShell (BYOA) Cloud agent platform Bare host / VPC

Read top to bottom. Layer 1 (adopt a product runtime) and Layer 2 (build with a framework) are alternatives for most teams — not two products you must stack. Do not compare chips across different rows as if they were the same decision.

Layer 0

Control pattern

Workflow · hybrid · autonomous loop · always-on presence

Layer 1

Product runtime

OpenClaw · Hermes · Deep Agents Code (NemoClaw) · other assistants

Layer 2

Build framework / BYOA

LangGraph · Google ADK · CrewAI · your image — enterprise-common

Layer 3

Execution platform

NemoClaw · OpenShell (BYOA) · cloud agent platform · bare host

Rule: There is no single official matrix that ranks all these names together. Good decisions pick one choice per layer in order. Bad decisions force a false either/or across layers (e.g. “LangGraph vs OpenClaw”).
02 · Decision flowchart

See the path, then read the detail

Do not start with a brand. Start with constraints that eliminate whole classes of solutions. Each step narrows the next.

Start / preferred pattern Decision question Candidate class Higher autonomy cost

This is a class selector, not a vendor ranking. Org allowlists and cloud mandates can force a branch even when another chip looks “better” on features.

1

Can you list the main steps up front?

If you can write the steps down ahead of time, prefer a deterministic workflow or cron pipeline. Reserve agent loops for branches you cannot hardcode. Most production systems are hybrids: workflow shell, agent nodes only where uncertainty is real.

2

Does it need unattended presence?

Schedules, messaging reachability, and multi-day memory point to an always-on assistant product or a long-running service you operate. One-shot CLI sessions are a different product shape.

3

Build vs adopt a runtime?

Need chat channels and an assistant UX quickly → evaluate OpenClaw or Hermes (if approved), packaged via NemoClaw when you want OpenShell sandboxing. Need graph control and app-owned orchestration → LangGraph, Google ADK, or other BYOA (your image + OpenShell policy). Coding CLI via NemoClaw → Deep Agents Code.

4

What does org policy allow?

Runtime allowlists, cloud mandates, and data residency beat feature matrices. An unapproved OpenClaw is not fixed by NemoClaw packaging. A GCP mandate is a heuristic toward ADK — not a law. Multi-cloud teams often keep LangGraph or BYOA.

5

What isolation boundary do you need?

Credential proxying, egress allowlists, and sandbox process limits are platform concerns. NemoClaw is the NVIDIA reference path for OpenClaw, Hermes, and Deep Agents Code on OpenShell. Custom / framework agents use OpenShell directly (openshell sandbox create --from …) — BYOA — not a NemoClaw blueprint. There is no Community --from openclaw path today.

    06 · Failure modes

    Decisions that look smart and fail

    Anti-pattern

    “Pick the most autonomous agent”

    Autonomy is a cost. If the path is known, a workflow with an LLM drafting step will usually be cheaper, safer, and easier to audit.

    Anti-pattern

    “LangGraph vs OpenClaw”

    Category error. One is a build framework; the other is an always-on product runtime. You can use both in one architecture — or neither.

    Anti-pattern

    “Sandbox makes unapproved software approved”

    Isolation reduces blast radius. It does not replace legal, security, or procurement approval for the agent runtime itself.

    Anti-pattern

    “Feature checklist as strategy”

    Channel count or star count can mislead. Dominant constraints — allowlist, cloud, auditability, unattended presence — should decide first.
