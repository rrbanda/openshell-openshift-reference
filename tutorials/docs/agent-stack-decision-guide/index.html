Choose the right agent stack

OpenClaw, Hermes, Google ADK, LangGraph, and “autonomous agents” are not rivals on one axis. Use this guide in order: decide the control pattern, then the product runtime or build framework, then isolation. Wrong layer comparisons produce wrong picks.

01 · Mental model

Four layers, four decisions

Treat each option as belonging to a layer. A complete system often combines one choice from several layers. Comparing LangGraph to OpenClaw without saying which layer you are deciding is the most common failure mode.

Layer 0
Pick one pattern Workflow Hybrid Autonomous loop Always-on presence
THEN COMPOSE ↓
Layer 1 or 2
Usually one path OpenClaw Hermes or LangGraph Google ADK
THEN HOST ON ↓
Layer 3
Pick isolation NemoClaw OpenShell alone Cloud agent platform Bare host / VPC

Read top to bottom: choose how control works, then whether you adopt a runtime or build with a framework, then where it runs. Do not compare chips across different rows as if they were the same decision.

Rule: There is no single official matrix that ranks all five names together. Good decisions compose layers. Bad decisions force a false either/or.
02 · Decision flowchart

See the path, then read the detail

Do not start with a brand. Start with constraints that eliminate whole classes of solutions. Each step narrows the next.

Start / preferred pattern Decision question Candidate class Higher autonomy cost

This is a class selector, not a vendor ranking. Org allowlists and cloud mandates can force a branch even when another chip looks “better” on features.

1

Is the path enumerable?

If steps are known ahead of time, prefer a deterministic workflow or cron pipeline. Reserve agent loops for branches you cannot hardcode. Most production systems are hybrids: workflow shell, agent nodes only where uncertainty is real.

2

Does it need unattended presence?

Schedules, messaging reachability, and multi-day memory point to an always-on assistant product or a long-running service you operate. One-shot CLI sessions are a different product shape.

3

Build vs adopt a runtime?

Need chat channels and an assistant UX quickly → evaluate OpenClaw or Hermes (if approved). Need graph control, checkpoints, and app-owned orchestration → evaluate LangGraph or Google ADK.

4

What does org policy allow?

Runtime allowlists, cloud mandates, and data residency beat feature matrices. An unapproved OpenClaw is not fixed by NemoClaw packaging. A GCP mandate pushes toward ADK even if LangGraph is more flexible.

5

What isolation boundary do you need?

Credential proxying, egress allowlists, and sandbox process limits are platform concerns. NemoClaw is the NVIDIA reference path for OpenClaw/Hermes on OpenShell. Custom agents usually need OpenShell-alone or another sandbox you integrate.

    05 · Failure modes

    Decisions that look smart and fail

    Anti-pattern

    “Pick the most autonomous agent”

    Autonomy is a cost. If the path is known, a workflow with an LLM drafting step will usually be cheaper, safer, and easier to audit.

    Anti-pattern

    “LangGraph vs OpenClaw”

    Category error. One is a build framework; the other is an always-on product runtime. You can use both in one architecture — or neither.

    Anti-pattern

    “Sandbox makes unapproved software approved”

    Isolation reduces blast radius. It does not replace legal, security, or procurement approval for the agent runtime itself.

    Anti-pattern

    “Feature checklist as strategy”

    Channel count or star count can mislead. Dominant constraints — allowlist, cloud, auditability, unattended presence — should decide first.
